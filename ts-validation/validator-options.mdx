## Validator Options

The validator in this library provides customizable behavior through the `options` object, which can be passed during initialization in the `options` property of [`v.validator()`](/ts-validation/v-validator).

### Available Options:


| Option      | Default    | Description                                                                                                                                                                                                                   |
|-------------|------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| dev         | `false`    | Enables debug mode when set to `true`. In debug mode, the validator logs detailed messages to the console to assist with troubleshooting.                                                                                     |
| resultsType | `"object"` | Determines the structure used for storing nested validation results:<br/> <code>"array"</code>: Nested validation results are stored in arrays. <br/> <code>"object"</code>: Nested validation results are stored in objects. |

<Callout type="warning">
    <em>Note:</em> The <code>resultsType</code> option has been added to support projects using multiple programming languages (not just TypeScript). It may be removed in future versions once other language versions (such as Java) are stabilized.
</Callout>

<Callout type="info">
    <em>Recommendation:</em> If you are working exclusively with TypeScript, we recommend using <code>"object"</code> for <code>resultsType</code>.
</Callout>
