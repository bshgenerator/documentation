# Plugins Commands

The `plugins` command enables to manage [BSH Engine plugins](/docs/bsh-engine/packaging/plugins) from the command line.

## `generate`

Generate a plugin from a template.

<Tabs items={['npm', 'npx']}>
    <Tabs.Tab title="npm">
    ```bash
    bshg plugins generate <plugin-dir> [options]
    # or
    bshg p g <plugin-dir> [options]
    ```
    </Tabs.Tab>
    <Tabs.Tab title="npx">
    ```bash
    npx @bshsolutions/cli plugins generate <plugin-dir> [options]
    # or
    npx @bshsolutions/cli p g <plugin-dir> [options]
    ```
    </Tabs.Tab>
</Tabs>

| Option | Description | Required | Default |
|--------|-------------|----------|---------|
| `-n, --name <name>` | Name of the plugin | no | `bshplugin` |
| `-p, --path <path>` | Path to the plugin directory | no | ./ |
| `-t, --template <template>` | Template to use for the plugin | no | [bshengine.plugin.template](https://github.com/bshgenerator/bshengine.plugin.template)|
| `-c, --clean` | Remove samples files | no | false |

<Callout type="default">
    `-template none` allows you to generate a minimal plugin (only `bshplugin.json` file).
</Callout>

### `manifest`

Generate a new manifest directory<br/>
this cmd will generate a new directory with specified name and will create a `__manifest__.json` file inside it.

```json {filename: "__manifest__.json"}
{
  "target": "<target-entity>",
  "dependencies": [
    "BshEntities"
  ]
}
```

<Tabs items={['npm', 'npx']}>
    <Tabs.Tab title="npm">
    ```bash
    bshg plugins generate manifest <name>
    # or
    bshg p g m <name>
    ```
    </Tabs.Tab>
    <Tabs.Tab title="npx">
    ```bash
    npx @bshsolutions/cli plugins generate manifest <name>
    # or
    npx @bshsolutions/cli p g m <name>
    ```
    </Tabs.Tab>
</Tabs>

| Argument | Description | Required | Default |
|----------|-------------|----------|---------|
| `<name>` | Name of the manifest directory | yes | ~ |

| Option | Description | Required | Default |
|--------|-------------|----------|---------|
| `-t, --target <path>` | Target Entity | no | `name` argument |
| `-d, --dependencies <dependencies>` | Dependencies | no | `BshEntities` |

### `content`

Generate a new content file inside a manifest directory

<Tabs items={['npm', 'npx']}>
    <Tabs.Tab title="npm">
    ```bash
    bshg plugins generate content <name> -m <manifest>
    # or
    bshg p g c <name> -m <manifest>
    ```
    </Tabs.Tab>
    <Tabs.Tab title="npx">
    ```bash
    npx @bshsolutions/cli plugins generate content <name> -m <manifest>
    # or
    npx @bshsolutions/cli p g c <name> -m <manifest>
    ```
    </Tabs.Tab>
</Tabs>

| Argument | Description | Required | Default |
|----------|-------------|----------|---------|
| `<name>` | Name of the content file | yes | ~ |

| Option | Description | Required | Default |
|--------|-------------|----------|---------|
| `-m, --manifest <manifest>` | Manifest directory name where the file will be created | yes | ~ |
| `-e, --entity <entity>` | Entity name to get the JSON structure from | no | `manifest` option value |
| `-o, --override` | Override existing content file if it exists | no | false |
| `--skipprompts` | Skip interactive prompts and use default values only | no | false |
| `-h, --help` | display help for command | no | false |

<Callout type='info'>
When creating a content file, the cmd will ask you for some content of the file, you can skip this by using the `--skipprompts` option.
</Callout>

<Callout type=''>
  Available entities to use with `--entity` option:
  - BshEntities
  - BshSchemas
  - BshTypes
  - BshRoles
  - BshPolicies
  - BshTriggers
  - BshEmailTemplates
</Callout>


## `install`

Install a plugin from a directory.

<Tabs items={['npm', 'npx']}>
    <Tabs.Tab title="npm">
    ```bash
    bshg plugins install <plugin-dir> [options]
    # or
    bshg p i <plugin-dir> [options]
    ```
    </Tabs.Tab>
    <Tabs.Tab title="npx">
    ```bash
    npx @bshsolutions/cli plugins install <plugin-dir> [options]
    # or
    npx @bshsolutions/cli p i <plugin-dir> [options]
    ```
    </Tabs.Tab>
</Tabs>

| Argument | Description | Required | Default |
|----------|-------------|----------|---------|
| `<plugin-dir>` | Path to the plugin directory | yes | ~ |

| Option | Description | Required | Default |
|--------|-------------|----------|---------|
| `-h, --host <host>` | BSH Engine host URL | yes | ~ |
| `-k, --api-key <key>` | BSH Engine API key | yes | ~ |
| `--no-verbose` | Disable verbose output (default: verbose enabled) | no | false |

**Usage Example:**

<Tabs items={['npm', 'npx']}>
    <Tabs.Tab title="npm">
    ```bash
    bshg plugins install ./path/to/plugin \
      --host https://instance-host.com \
      --api-key <api-key>
    ```
    </Tabs.Tab>
    <Tabs.Tab title="npx">
    ```bash
    npx @bshsolutions/cli plugins install ./path/to/plugin \
      --host https://instance-host.com \
      --api-key <api-key>
    ```
    </Tabs.Tab>
</Tabs>
