# Mailing API

The Mailing API provides comprehensive email management operations including sending emails, managing email templates, configuring SMTP settings, and tracking email delivery status.

<Callout type="info">
  This API is linked with the `BshEmails` entity for email tracking and logging.
</Callout>

## Base Endpoint

```bash
/api/mailing
```

## Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| [POST](/docs/bsh-engine/api/mailing/send-email) | `/api/mailing/send` | Send an email using plain text, HTML, or templates |
| [GET](/docs/bsh-engine/api/mailing/configurations) | `/api/mailing/configurations` | Get email configuration settings |
| [POST](/docs/bsh-engine/api/mailing/configurations) | `/api/mailing/configurations` | Update email configuration settings |
| [GET](/docs/bsh-engine/api/mailing/templates) | `/api/mailing/templates` | Get available email templates |
| [POST](/docs/bsh-engine/api/mailing/templates) | `/api/mailing/templates` | Create or update email templates |
| [GET](/docs/bsh-engine/api/mailing/logs) | `/api/mailing/logs` | Get email delivery logs and statistics |

## Security

All Mailing API endpoints are secured by the [authentication](/docs/bsh-engine/api/authentication) system. Email operations require proper authentication tokens and users must have appropriate permissions for email management. Unauthorized requests will be denied accordingly.
