## Response Format

All API responses follow a consistent JSON format:

```json {2}
{
  "data": [],
  "timestamp": 1234567890,
  "code": 200,
  "status": "OK",
  "meta": {},
}
```

<Callout type='info'>
  The egnine always returns a **list** of items, even if there is only one item.
</Callout>

## Pagination

Pagination is supported for all endpoints that return a list of items.

```json {7-13}
{
  "data": [],
  "timestamp": 1234567890,
  "code": 200,
  "status": "OK",
  "meta": {},
  "pagination": {
    "current": 0,
    "total": 20,
    "pages": 2,
    "first": true,
    "last": false
  }
}
```

## Error Handling

Errors are returned with appropriate HTTP status codes and error details:

```json {7,8,10}
{
  "data": [],
  "timestamp": 1234567890,
  "code": 400,
  "status": "ERROR STATUS",
  "meta": {
    "sql": "<SQL query>",
    "tips": {}
  },
  "error": "<Error message>"
}
```

the `meta.sql` is the SQL query that was executed to get the error.<br/>
the `meta.tips` is a list of tips to help you fix the error.

## Validation Errors

Validation errors are returned as an object with the field name and the error message.<br/>
this error is offen sent when you try to insert or update data with invalid data.

```json {11-16}
{
  "data": [],
  "timestamp": 1234567890,
  "code": 400,
  "status": "ERROR STATUS",
  "meta": {
    "sql": "<SQL query>",
    "tips": {},
  },
  "error": "<Error message>",
  "validations": [
    {
      "field": "<Field name>",
      "error": "<Error message>"
    }
  ]
}
```
